# Glob 通配（文件名模式）教程

glob 是 Shell 中用于“路径匹配”的模式语法，常用于匹配文件和目录。

> glob 不是正则表达式。它更简单，主要面向文件路径。

---

## 1. 常见符号

| 模式 | 含义 | 示例 |
|---|---|---|
| `*` | 匹配任意长度字符（不含路径分隔符 `/`） | `*.md` |
| `?` | 匹配任意单个字符 | `file?.txt` |
| `[...]` | 匹配字符集合中的一个字符 | `file[12].txt` |
| `[!...]` 或 `[^...]` | 匹配不在集合中的一个字符（依 shell 而异） | `file[!0-9].txt` |
| `**` | 递归匹配多级目录（需 shell/工具支持） | `**/*.md` |
| `{a,b}` | 大括号展开（严格说是 brace expansion） | `*.{jpg,png}` |

---

## 2. 典型示例

```bash
# 当前目录下所有 markdown 文件
ls *.md

# 匹配 file1.txt 到 file9.txt
ls file?.txt

# 匹配 a.txt 或 b.txt
ls [ab].txt

# 递归查找所有 .md（bash 需开启 globstar）
shopt -s globstar
ls **/*.md

# 多后缀匹配
ls *.{js,ts,tsx}
```

---

## 3. glob 与 regex 的区别

- glob：主要匹配**文件路径**，语法简单，常在 shell、构建工具配置中使用。
- regex：匹配**任意文本内容**，表达能力更强。

对比示例：
- glob：`*.md`
- regex：`^.*\.md$`

---

## 4. 常见场景

- 命令行文件筛选：`cp *.log backup/`
- 工具配置：`eslint "src/**/*.ts"`
- 批处理：`rm temp-*.tmp`

---

## 5. 易错点

- `*` 不会跨目录（`/`）；跨目录通常要用 `**`。
- 不同 shell、工具对 `**` 和 `[!...]` 支持不同。
- 当没有匹配项时，有些 shell 会保留原样字符串。

---

## 6. 学习建议

- 先熟悉 `* ? []` 三件套。
- 递归匹配时确认工具是否支持 `**`。
- 在危险命令（如 `rm`）前先用 `echo` 预演：

```bash
echo rm *.tmp
```
