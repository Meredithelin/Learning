# GCC 编译器使用手册 (GCC Usage Guide)

## 0. 背景与起源

### 0.1 GCC 的诞生
GCC (GNU Compiler Collection) 的历史可以追溯到 1985 年。
- **创始人**：由自由软件基金会 (FSF) 的创始人 **Richard Stallman** 发起。
- **初衷**：当时 Unix 系统上的 C 编译器都是商业收费软件。为了构建完全自由的 GNU 操作系统，Stallman 需要一个自由的编译器。
- **演进**：最初 GCC 代表 "GNU C Compiler"，仅支持 C 语言。随着支持的语言（C++、Fortran、Go 等）增多，它更名为 "GNU Compiler Collection"。它是自由软件运动的基石，也是 Linux 内核及大多数开源项目的核心工具。

### 0.2 类似编译器的起源
- **Clang/LLVM**：起源于 2003 年伊利诺伊大学的一个研究项目（由 Chris Lattner 领导）。后来苹果公司 (Apple) 大力资助并采用它来替代 GCC。其设计目标是更模块化、编译速度更快、错误提示更友好，且采用更宽松的 BSD 许可证。
- **MSVC (Microsoft Visual C++)**：源自微软 1993 年发布的 Visual C++ 1.0。它是 Windows 平台上的标准编译器，紧密集成在 Visual Studio IDE 中，其前身可以追溯到 80 年代的 Microsoft C 编译器。
- **PCC (Portable C Compiler)**：由 Stephen C. Johnson 在 70 年代中后期于贝尔实验室开发。它是历史上第一个真正“可移植”的编译器，极大地推动了 Unix 在不同硬件架构上的普及，也是 GCC 诞生前最流行的编译器。
- **ICC/ICX (Intel Compiler)**：英特尔开发的针对其自家处理器高度优化的编译器，现已转向基于 LLVM 架构。

## 1. 基础编译流程
### 1.1 最简编译
将 `main.c` 编译为可执行文件（默认为 `a.out`）：
```bash
gcc main.c
./a.out
```

### 1.2 指定输出名称
使用 `-o` 选项指定生成的可执行文件名：
```bash
gcc main.c -o my_program
./my_program
```

## 2. 编译的四个阶段
GCC 编译过程通常分为四个阶段，可以使用特定选项停在某一阶段：

| 阶段 | 说明 | 命令示例 | 生成文件 |
| :--- | :--- | :--- | :--- |
| **预处理** | 处理宏定义、头文件包含 | `gcc -E main.c -o main.i` | `.i` (预处理文件) |
| **编译** | 将代码转换为汇编代码 | `gcc -S main.i -o main.s` | `.s` (汇编文件) |
| **汇编** | 将汇编转换为机器码 | `gcc -c main.s -o main.o` | `.o` (目标文件) |
| **链接** | 合并目标文件与库文件 | `gcc main.o -o main` | 无后缀 (可执行文件) |

## 3. 常用编译选项
### 3.1 警告与错误处理
- `-Wall` - 开启大部分常用的警告信息（强烈建议始终开启）。
- `-Wextra` - 开启额外的警告信息。
- `-Werror` - 将所有警告视为错误，强制停止编译。
- `-w` - 关闭所有警告（不推荐）。

### 3.2 调试与优化
- `-g` - 在可执行文件中包含调试信息，以便使用 `gdb` 调试。
- `-O0` - 不进行优化（默认，适合调试）。
- `-O2` - 常用优化级别，平衡编译速度和执行效率。
- `-O3` - 最高级别优化，可能增加二进制文件体积。
- `-Os` - 专门针对文件大小进行优化。

### 3.3 标准选择
- `-std=c99` - 使用 C99 标准。
- `-std=c11` - 使用 C11 标准。
- `-std=c++17` - 使用 C++17 标准（需使用 `g++`）。

## 4. 多文件编译与库管理
### 4.1 编译多个源文件
```bash
gcc main.c tool.c -o my_app
```

### 4.2 包含头文件与库
- `-I <dir>` - 添加头文件搜索路径。
- `-L <dir>` - 添加库文件搜索路径。
- `-l<libname>` - 链接指定的库（如 `-lm` 链接数学库 `libm.so`）。

```bash
gcc main.c -I./include -L./lib -lmysqlclient -o db_tool
```

### 4.3 静态与动态链接
- 默认情况下，GCC 优先使用动态链接。
- `-static` - 强制进行静态链接，生成的文件不依赖外部库，但体积较大。

## 5. C++ 编译
虽然 `gcc` 可以编译 C++ 文件，但通常建议使用 `g++`，因为它会自动链接 C++ 标准库：
```bash
g++ main.cpp -o my_cpp_app
```

## 6. 常见技巧与最佳实践
- **Makefile/CMake**：当项目包含超过 3 个文件时，建议使用构建工具。
- **并行编译**：在使用 `make` 时，使用 `make -j$(nproc)` 加快编译速度。
- **检查依赖**：使用 `ldd <program>` 查看程序依赖的动态库。
- **推荐配置**：日常开发建议使用 `gcc -Wall -Wextra -g -o output input.c`。
