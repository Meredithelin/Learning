# 正则表达式（Regex）使用教程

正则表达式是一种“文本匹配规则语言”，用于查找、过滤、替换字符串。

---

## 1. 常见用途

- 搜索日志中的错误行
- 校验输入格式（邮箱、手机号等）
- 从文本中提取目标字段
- 批量替换内容

---

## 2. 基础语法速查

| 语法 | 含义 | 示例 |
|---|---|---|
| `.` | 任意单个字符（通常不含换行） | `a.c` 匹配 `abc` |
| `*` | 前一个元素重复 0 次或多次 | `ab*` 匹配 `a`, `ab`, `abbb` |
| `+` | 前一个元素重复 1 次或多次 | `ab+` 匹配 `ab`, `abbb` |
| `?` | 前一个元素重复 0 次或 1 次 | `colou?r` 匹配 `color`, `colour` |
| `[]` | 字符集合 | `[abc]` 匹配 `a/b/c` |
| `[^]` | 否定字符集合 | `[^0-9]` 匹配非数字 |
| `{n}` | 重复 n 次 | `\d{4}` |
| `{n,}` | 至少重复 n 次 | `\w{3,}` |
| `{n,m}` | 重复 n 到 m 次 | `\d{2,4}` |
| `^` | 行首锚点 | `^ERROR` |
| `$` | 行尾锚点 | `\.log$` |
| `()` | 分组 | `(ab)+` |
| `\|` | 或 | `cat\|dog` |
| `\d` | 数字（0-9） | `\d+` |
| `\w` | 单词字符（字母数字下划线） | `\w+` |
| `\s` | 空白字符 | `\s+` |

> 在大多数语言中，反斜杠需要转义，例如字符串里常写成 `"\\d+"`。

---

## 3. 经典示例

### 3.1 匹配邮箱（简化版）

```regex
^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}$
```

### 3.2 匹配 11 位手机号（中国大陆简化）

```regex
^1[3-9]\d{9}$
```

### 3.3 匹配日期 `YYYY-MM-DD`

```regex
^\d{4}-(0[1-9]|1[0-2])-(0[1-9]|[12]\d|3[01])$
```

---

## 4. grep 中使用 Regex

```bash
# 查找以 ERROR 开头的行
grep -E '^ERROR' app.log

# 查找邮箱样式文本
grep -E '[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}' users.txt

# 统计匹配行数
grep -Ec 'timeout|failed' app.log
```

---

## 5. 易错点

- `*` 只作用于“前一个元素”，不是整个词。
- 贪婪匹配可能会“吃太多”，需要时使用非贪婪（如 `.*?`，取决于具体引擎）。
- 不同工具/语言支持的语法略有差异（POSIX、PCRE、JavaScript 等）。

---

## 6. 学习建议

- 先掌握：字符类、重复、锚点、分组、或逻辑。
- 每写一个表达式都用样例正反测试。
- 从“小规则”组合成“大规则”，不要一开始写超长表达式。
