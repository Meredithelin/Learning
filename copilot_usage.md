# GitHub Copilot 使用指南

GitHub Copilot 是一款由 AI 驱动的编程助手，能够实时为你提供代码建议和自动补全。它像是一个随时待命的资深开发者，协助你编写代码、测试和文档。

## 1. 核心功能

### 1.1 代码自动补全 (Inline Suggestions)
当你开始编写代码时，Copilot 会根据上下文自动给出灰色半透明的建议。
- **接受建议**：按 `Tab` 键。
- **接受部分建议**：按 `Ctrl` + `→` (Windows/Linux) 或 `Cmd` + `→` (macOS) 逐词接受。
- **查看下一个/上一个建议**：按 `Alt` + `]` 或 `Alt` + `[`。
- **拒绝建议**：继续输入或按 `Esc`。

### 1.2 聊天对话 (GitHub Copilot Chat)
在侧边栏点击 Copilot 图标，或者在编辑器中按 `Ctrl` + `I` (Windows/Linux) 或 `Cmd` + `I` (macOS) 唤起内联聊天。
- **代码解释**：选中一段代码，输入 `/explain`，它会逐行解析逻辑。
- **修复 Bug**：输入 `/fix`，它会分析错误并给出修复方案。
- **编写测试**：输入 `/tests`，它会为当前函数生成单元测试。
- **优化代码**：输入 `/optimize`，它会提供性能更好的写法。

### 1.3 自然语言转代码 (Comments to Code)
写一段注释，描述你想实现的功能，然后按回车。
```javascript
// 写一个函数，计算两个日期之间的天数
function 
```
Copilot 会自动补全函数体。

## 2. 常用斜杠命令 (Slash Commands)
在 Copilot Chat 中使用斜杠命令可以更精确地执行特定任务：
- `/help`：获取关于 GitHub Copilot 的帮助。
- `/doc`：为选中的代码添加文档注释（如 JSDoc）。
- `/clear`：清空当前的聊天会话。
- `/workspace`：在整个工作区搜索并回答问题。

## 3. 最佳实践 (Tips for Success)

### 3.1 给它清晰的上下文
Copilot 就像“读心术”，但它需要上下文。如果你打开了相关的多个文件，它的预测会更准确。
- **保持相关文件打开**：Copilot 会参考你当前打开的其他标签页。
- **编写有意义的函数名和变量名**：清晰的命名有助于它理解你的意图。

### 3.2 编写高质量的注释
越具体越好。
- *坏例子*：`// 排序数组`
- *好例子*：`// 使用快速排序算法对包含对象的数组按 'id' 属性进行升序排列`

### 3.3 验证 AI 的结果
**记住：AI 也会写出错误的或有安全漏洞的代码。**
- 始终运行测试。
- 像对待初级实习生的代码一样进行 Code Review。
